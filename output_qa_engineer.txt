## Workflow Overview

**Step 1: Receive Code Summary**
- The Developer provides an overview of the project including key features,
features, models, endpoints, and any notable challenges or issues encounter
encountered.

**Step 2: Run the Project**
- Navigate to the project directory:
  ```bash
  cd e:/Society_Managment/erp_project
  ```
- Build and start the project:
  ```bash
  npm run build && npm start
  ```

**Step 3: Execute Integration Tests**
- Run integration tests using the following command:
  ```bash
  npm test
  ```

**Step 4: Perform Static Analysis**
- Lint the code for any issues and formatting problems:
  - **Linting:**
    ```bash
    npm run lint
    ```
  - **Format Check (optional):**
    ```bash
    npm run format --check
    ```

**Step 5: Measure Performance**
- Measure request-response timings to evaluate performance. This can be don
done using tools like `curl` or any web-based testing tool.

**Step 6: Compile QA Report**
- **Test Results:** Document the outcome of integration tests (pass/fail).
- **Lint/Format Issues:** List all issues found during static analysis.
- **Suggested Fixes/Optimizations:** Propose improvements or bug fixes base
based on findings.

**Step 7: Create Git Branch and Commit Changes**
- Automatically create a `qa-fixes` branch if needed:
  ```bash
  git checkout -b qa-fixes
  ```
- Apply suggested changes in this branch, commit them with descriptive mess
messages:
  ```bash
  git add .
  git commit -m "Apply suggested fixes from QA review"
  ```

**Step 8: Pass the QA Report Back**
- Share the QA report and any related files or screenshots back to the Deve
Developer.

**Step 9 (Optional): Store Reports in Memory DB**
- If a memory database is available, store the QA reports for future refere
reference.
