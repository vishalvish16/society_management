Agent 2 — Code Review

Review Agent

Reviews all code for correctness, architecture quality, and spec compliance

Backend checks
SaaS Multi-tenancy: society_id check on every query — NO cross-society data leaks
Plan Enforcement: checkPlanLimit middleware is correctly applied to restricted routes
Role Guards: role-based hierarchy (SA > Pramukh > Sec > Resident > Watchman) is enforced
Async wrappers used on all Express routes (no unhandled promise rejections)
Response Consistency: all endpoints return { success, data, message } envelope
Prisma optimization: selective fetching (select/include) to avoid heavy payloads
Critical Indexes: unit.society_id, bill.unit_id + billing_month, visitor.qr_token, user.phone
Flutter checks
Layout Consistency: ResponsiveBuilder used for 768px breakpoint (Web table vs Mobile cards)
State Management: Riverpod used correctly with repositories injected (no direct API in widgets)
Pagination: server-side page-based for web, cursor-based infinite scroll for mobile
UX/UI Patterns: Shimmer loaders, empty state illustrations, and correct keyboardTypes
File Handling: multi_file_attachments display and PDF receipt viewer integration
Architecture checks
Module boundary compliance: no logic crossing feature modules without services
No hardcoded configuration: all values sourced from .env or DB settings table

Agent 3 — Security

Security Agent

Audits the codebase and infrastructure for vulnerabilities and hardening gaps

Auth & tokens
JWT uses RS256 (asymmetric) — reject if HS256 is used
Access token expiry ≤ 15 min — reject if longer
Refresh token in httpOnly + Secure + SameSite=Strict cookie only
Refresh token stored in DB — revocation must be possible
Rate limit on /auth/login: max 5 attempts / 15 min / IP
bcrypt cost factor ≥ 12
API & input
Zod schema validates every request body and query param
All DB queries via Prisma parameterised — no raw SQL string concat
File uploads: MIME type validated, max 10 MB enforced, extension allowlist
CORS restricted to app bundle ID and web domain only
Helmet.js applied with full default headers on all routes
QR & visitor flow
QR token is UUID — not sequential or guessable
Redis TTL enforced on every QR token — no fallback bypass
One-time use: token deleted from Redis on first valid scan
Validate endpoint checks society_id of watchman vs visitor
Infrastructure
No secrets in source code or Docker images — .env only
S3 bucket not publicly accessible — only via presigned URLs
PostgreSQL not exposed externally — only accessible from backend container
Redis not exposed externally — password-protected
flutter_secure_storage used for all tokens on mobile — no SharedPreferences

npm auditOWASP checklistdependency-checktrivy (Docker)
Outputs
Security audit report Severity-ranked issue list Hardening patches
Triggered by: Dev Agent output (parallel to Review Agent). Blocks deployment until all HIGH issues resolved.
