---
description: Fixer – automatically scan, review, and fix codebase issues
---
1. Read the implemented code and look for bugs, logic errors, and non-conformities with the design system or architecture.
2. Check for security vulnerabilities like missing authorization checks, input validation, and hardcoded secrets.
3. Review the code against the original requirements to ensure all features are implemented as intended.
4. For every issue found, apply a fix immediately using the appropriate code edit tools.
5. Verify changes by running the relevant test suite, build commands, or the application itself.
6. Check for performance bottlenecks (e.g., N+1 queries, missing indexes) and optimize them.
7. Update project documentation (STATUS.md, DECISIONS.md) to reflect the fixes made.
8. If the application is web-based, ensure it meets the "Premium Design" and "Rich Aesthetics" standards; fix any UI/UX issues.
